Actually, nothing much except for few nice properties. Lets dissect this :

Sigmoid unit :
()=11+()f(x)=11+exp(x)

Tanh unit:
()=()f(x)=tanh(x)

Rectified linear unit (ReLU):
()=inf=1(+0.5)(1+)f(x)=i=1inf(xi+0.5)log(1+ex)

we refer

  • inf=1(+0.5)i=1inf(xi+0.5) as stepped sigmoid
  • (1+)log(1+ex) as softplus function

The softplus function can be approximated by max function (or hard max ) ie (0,+(0,1))max(0,x+N(0,1)) . The max function is commonly known as Rectified Linear Function (ReL).
 
In the following figure, we graphically compare ReL function (soft/hard) with sigmoid function.

The major differences between the sigmoid and ReL function are:

  • Sigmoid function has range [0,1] whereas the ReL function has range [0,][0,]. Hence sigmoid function can be used to model probability, whereas ReL can be used to model positive real number. NOTE: The view of softplus function as approximation of stepped sigmoid units relates to the binomial hidden units as discussed in http://machinelearning.wustl.edu...
  • The gradient of the sigmoid function vanishes as we increase or decrease x. However, the gradient of the ReL function doesn't vanish as we increase x. In fact, for max function, gradient is defined as 01  <0  >0{0if  x<01if  x>0} .

The advantages of using Rectified Linear Units in neural networks are

  • If hard max function is used as activation function, it induces the sparsity in the hidden units.
  • ReLU doesn't face gradient vanishing problem as with sigmoid and tanh function. Also, It has been shown that deep networks can be trained efficiently using ReLU even without pre-training.

References :

  • On Rectified Linear Units for Speech Processing http://www.cs.toronto.edu/~hinto...
  • Rectifier Nonlinearities Improve Neural Network Acoustic Models http://ai.stanford.edu/~amaas/pa...
  • Deep Sparse Rectifier Neural Networks

ReLU has two benefits 1) It is fast 2) It does not suffer from the vanishing gradient problem.

2) Has been covered by the answers below. I will just talk about why it is fast.

ReLU's are faster to compute because 1) They supposedly do not require any normalization. 2) They do not require any exponential computation (such as those required in sigmoid or tanh activations). The original Alexnet paper reported around 6X over existing activation functions[1].

However, this benefit did not last very long as practitioners figured ReLU suffers from the "Knockout Problem". With large learning rates, a large gradient can kill a ReLU such that it never gets activated again, ever. To fix this,  a good generalization is Maxout that caps the weight-data dot-product, saving it from the knock out problem[2]. It is worth noting that recent work on batch normalization suggests that perhaps we are not confined to just using ReLUs, and adding a linear layer before activation functions to perform normalization with batch-wise whitening and re-scaling, can avoid many of the problems that ReLUs solves[3].

[1] Alexnet : Page on toronto.edu
[2] Maxout :Page on mit.edu
[3] Batch Normalization: Page on arxiv.org

RELU were a MILESTONE because for first time, they made it so possible to directly train a DEEP Supervised NN i.e. without having to worry too much about super-clever initialization or unsupervised pre-training.

I think they were BREAKTHROUGH also because the idea that they should work so well seemed COUNTER-INTUITIVE at first. After all, they lacked many nice mathematical properties that were offered by previous activation functions like sigmoid & tanh:

  • Not Differentiable at 0
  • Derivative being 0 on left side (=> may prevent backprop-based learning and propagation of errors)
  • lacks symmetry / Anti-symmetry (=> may need twice as many neurons to encode full relationship)
  • is Unbounded on the right side (=> may cause Numeric instability during Optimization)
  • offers Hard non-linearity i.e. at just 1 point = 0, and seem much too simple/ linear elsewhere ( => may not seem like best way to impart non-linearity to Networks at first)

In Hindsight, there are multiple explanations on why RELU mechanism may work as well as it does in practice:

  • promotes Sparse Representations
  • enables learning Deep Nets (hence more rich/ abstract feature generators e.g. in CNN)
